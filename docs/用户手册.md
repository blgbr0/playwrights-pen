## 简介

PlaywrightsPen（剧作家之笔）是一款基于 **Playwright MCP** 的智能化自动化测试服务。它结合大语言模型（LLM）与浏览器自动化能力，让您可以用**自然语言**编写测试用例，无需了解复杂的选择器语法。

### 核心特性

| 特性 | 说明 |
|------|------|
| 🗣️ **自然语言测试** | 用中文或英文描述测试场景，系统自动解析执行 |
| 🔄 **交互式首轮执行** | 首次运行时支持人工确认关键步骤，确保测试正确性 |
| 🎯 **智能回归测试** | 基于首轮执行记录，自动化后续回归测试 |
| 🔒 **私有化部署** | 数据不外泄，支持任意 OpenAI 兼容的 LLM 模型 |

## 系统要求

- **操作系统**：Windows 10+、Linux、macOS
- **Python 版本**：3.10 或更高版本
- **Node.js**：16+ （用于 Playwright MCP）
- **网络要求**：需要能访问配置的 LLM API 服务

## 安装指南

### 1. 获取项目

```bash
# 进入项目目录
cd playwrights_pen
```

### 2. 创建虚拟环境

**使用 Conda (推荐)：**
```bash
# 创建名为 playwrights_pen 的环境，指定 Python 3.10 
conda create -n playwrights_pen python=3.10 -y

# 激活新环境
conda activate playwrights_pen
```

**使用 venv (备选)：**

*Windows：*
```bash
python -m venv .venv
.venv\Scripts\activate
```

*Linux/macOS：*
```bash
python -m venv .venv
source .venv/bin/activate
```

### 3. 安装依赖

```bash
# 安装项目及依赖
pip install -e .

# 安装 Playwright 浏览器（首次需要）
playwright install chromium
```

### 4. 验证安装

```bash
playwrights-pen --help
```

如果看到帮助信息，说明安装成功。

## 配置说明

### 1. 创建配置文件

```bash
# Windows
copy .env.example .env

# Linux/macOS
cp .env.example .env
```

### 2. 编辑配置

用编辑器打开 `.env` 文件，根据需要修改以下配置：

```env
# ========== LLM 配置 ==========
# LLM API 密钥（必填）
LLM_API_KEY=your-api-key-here

# LLM API 地址（支持 OpenAI 兼容接口）
# OpenAI: https://api.openai.com/v1
# DeepSeek: https://api.deepseek.com
# 本地 Ollama: http://localhost:11434/v1
LLM_BASE_URL=https://api.openai.com/v1

# 使用的模型名称
LLM_MODEL=gpt-4o

# ========== 浏览器配置 ==========
# 是否无头模式运行（true=不显示浏览器窗口）
BROWSER_HEADLESS=false

# ========== MCP 配置 ==========
# MCP 启动命令（一般无需修改）
MCP_COMMAND=npx
MCP_ARGS=@playwright/mcp@latest

# ========== 服务配置 ==========
# REST API 监听地址
HOST=0.0.0.0
PORT=8000

# ========== 存储配置 ==========
# 数据存储目录
DATA_DIR=./data
# PostgreSQL 连接（留空使用 SQLite）
DATABASE_URL=postgresql+asyncpg://user:pass@host:port/db

# ========== 执行配置 ==========
# 默认录制模式: none, minimal, standard (默认), full
DEFAULT_RECORDING_MODE=standard
# 默认数据输入模式: preset, prompt (默认), generate
DEFAULT_DATA_INPUT_MODE=prompt

# ========== Electron 配置 ==========
# 打包应用路径（.exe/.app/AppImage）
ELECTRON_EXECUTABLE_PATH=/path/to/app
# 开发项目路径
ELECTRON_PROJECT_PATH=/path/to/project
```

### 3. 常用 LLM 配置示例

**OpenAI：**
```env
LLM_API_KEY=sk-xxxxx
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o
```

**DeepSeek：**
```env

```

**本地 Ollama：**
```env
LLM_API_KEY=ollama
LLM_BASE_URL=http://localhost:11434/v1
LLM_MODEL=qwen2.5:7b
```

## 命令行使用

### 基本命令

#### 运行测试

```bash
playwrights-pen run "测试描述"
```

**示例：**
```bash
# 简单搜索测试
playwrights-pen run "打开百度，搜索Playwright，验证搜索结果页面包含Playwright"

# 表单填写测试
playwrights-pen run "打开示例网站，填写用户名admin和密码123456，点击登录按钮"
```

#### 指定确认模式

```bash
# 每步确认（适合首次探索）
playwrights-pen run "..." --confirm every_step

# 仅关键步骤确认（默认，推荐日常使用）
playwrights-pen run "..." --confirm key_steps

# 完全自动执行（适合回归测试）
playwrights-pen run "..." --confirm none
```

#### Electron 应用测试 (实操指南)

PlaywrightsPen 支持通过 CDP 节点对本地 Electron 应用直接进行端到端自动化测试。你可以直接通过我们提供的示例项目体验此功能：

**实战演练：测试内置待办事项 Demo**

1. **安装示例项目依赖**
   进入项目自带的 `examples/electron-demo` 目录并安装必要依赖：
   ```bash
   cd examples/electron-demo
   npm install
   ```

2. **执行第一条端到端用例**
   返回外层主目录（或在激活了环境的任意路径下），执行如下命令，让大模型控制该 Demo：
   ```bash
   playwrights-pen run-electron "在任务输入框输入学习 PlaywrightsPen，然后点击添加按钮" --app ./examples/electron-demo --dev
   ```

   *执行步骤拆解：*
   1. 程序会在后台临时启动 Electron 应用。
   2. 大模型会识别输入框，并触发注入属性（`data-ref`）。
   3. 在终端里，你会看到执行器展示出将要执行的物理动作。
   4. 输入 `y` 同意执行，你将能目睹输入框被填入文本并提交的界面交互动作。

**其他使用姿势：**

```bash
# 测试打包后的 Electron 应用
playwrights-pen run-electron "点击设置按钮，修改用户名" --app /path/to/app.exe

# 测试开发中的项目（自动检测 package.json）
playwrights-pen run-electron "验证主窗口显示" --app /path/to/project --dev

# 使用环境变量配置默认路径后直接运行
export ELECTRON_EXECUTABLE_PATH="/path/to/myapp.exe"
playwrights-pen run-electron "测试登录功能"
```

**参数说明：**
| 参数 | 说明 |
|------|------|
| `--app, -a` | Electron 可执行文件或项目所在目录的绝对/相对路径 |
| `--dev, -d` | 强制使用开发模式启动项目（即后台运行 `npm start`） |
| `--confirm, -c` | 确认模式：every_step / key_steps / none |

#### 指定执行模式与回放

```bash
# 探索模式（首次执行，LLM 实时推理）
playwrights-pen run "..." --mode exploration

# 回归模式（基于历史记录搜索元素）
playwrights-pen run "..." --mode regression

# 直接回放已被充分验证的用例（速度最快，自动生成单次详细报告）
# 此命令将完全使用已有录制的元素引用(recorded_ref)执行
playwrights-pen replay <test_case_id>

# 指定以特定的会话记录作为回放基准
playwrights-pen replay <test_case_id> -s <session_id>
```

#### 无头模式

```bash
# 后台运行，不显示浏览器窗口
playwrights-pen run "..." --headless
```

### 管理命令

```bash
# 查看已保存的测试用例
playwrights-pen list-cases

# 查看执行会话历史
playwrights-pen list-sessions

# 按测试用例筛选会话
playwrights-pen list-sessions --case <test_case_id>

# 查看当前配置
playwrights-pen config
```

### 启动 API 服务器

```bash
# 使用默认配置启动
playwrights-pen serve

# 指定端口
playwrights-pen serve --port 9000

# 禁用自动重载（生产环境）
playwrights-pen serve --no-reload
```

### 批量执行与测试套件

按标签批量运行测试用例：

```bash
# 运行所有带 @smoke 标签的用例
playwrights-pen run-suite @smoke

# 运行多个标签（OR 逻辑）
playwrights-pen run-suite @login @order

# 失败自动重试 2 次
playwrights-pen run-suite @smoke --retry 2

# 遇到失败立即停止
playwrights-pen run-suite @smoke --stop-on-fail

# 输出 JSON 格式
playwrights-pen run-suite @smoke --output json

# 同时生成 HTML 报告
playwrights-pen run-suite @smoke --report
```

**快捷命令：**
```bash
# 快速冒烟测试（等同于 run-suite @smoke）
playwrights-pen smoke
```

### 测试报告

```bash
# 生成最近 7 天的 HTML 报告
playwrights-pen report

# 生成最近 30 天的报告
playwrights-pen report --days 30
```

## REST API 使用

### 启动服务

```bash
playwrights-pen serve
```

服务启动后，可访问：
- **API 文档**：http://localhost:8000/docs（Swagger UI）
- **快速文档**：http://localhost:8000/redoc（ReDoc）

### API 端点

#### 测试用例管理

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/v1/testcases` | 创建测试用例 |
| GET | `/api/v1/testcases` | 列出所有测试用例 |
| GET | `/api/v1/testcases/{id}` | 获取测试用例详情 |
| PUT | `/api/v1/testcases/{id}` | 更新测试用例 |
| DELETE | `/api/v1/testcases/{id}` | 删除测试用例 |
| POST | `/api/v1/testcases/{id}/parse` | 重新解析测试用例的自然语言描述 |

#### 执行会话管理

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/v1/sessions` | 创建并执行测试会话 |
| GET | `/api/v1/sessions` | 列出会话 |
| GET | `/api/v1/sessions/{id}` | 获取会话详情 |
| POST | `/api/v1/sessions/{id}/confirm` | 确认当前待确认的步骤 |
| POST | `/api/v1/sessions/{id}/abort` | 中止正在运行的会话 |
| DELETE | `/api/v1/sessions/{id}` | 删除会话记录 |

#### 执行历史与统计

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/v1/sessions/history/stats` | 执行统计（通过率、平均时长等） |
| GET | `/api/v1/sessions/history/recent` | 最近执行记录 |
| GET | `/api/v1/sessions/history/trends` | 执行趋势分析（按天） |
| GET | `/api/v1/sessions/history/{id}/recordings` | 获取会话录制文件 |

### 示例请求

**创建测试用例：**
```bash
curl -X POST http://localhost:8000/api/v1/testcases \
  -H "Content-Type: application/json" \
  -d '{
    "name": "百度搜索测试",
    "description": "打开百度首页，搜索Playwright，验证结果包含playwright",
    "parse_now": true
  }'
```

**使用数据占位符：**
```bash
curl -X POST http://localhost:8000/api/v1/testcases \
  -H "Content-Type: application/json" \
  -d '{
    "name": "登录测试",
    "description": "打开登录页，输入用户名 {{username}}，输入密码 {{password}}，点击登录",
    "parse_now": true
  }'
```

> 💡 **占位符语法**：使用 `{{variable}}` 定义数据占位符，运行时会根据配置的数据模式（preset/prompt/generate）获取值。

**执行测试：**
```bash
curl -X POST http://localhost:8000/api/v1/sessions \
  -H "Content-Type: application/json" \
  -d '{
    "test_case_id": "your-test-case-id",
    "mode": "exploration",
    "confirmation_mode": "none"
  }'
```

**获取执行统计：**
```bash
curl http://localhost:8000/api/v1/sessions/history/stats
```

**获取执行趋势：**
```bash
curl "http://localhost:8000/api/v1/sessions/history/trends?days=7"
```

**查看测试用例列表：**
```bash
curl http://localhost:8000/api/v1/testcases
```

## 测试执行模式

### 确认模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `every_step` | 每个操作步骤都需要人工确认 | 首次探索新测试流程，需要仔细验证每一步 |
| `key_steps` | 仅关键步骤需要确认（如提交、删除操作） | 日常使用推荐，平衡效率与安全 |
| `none` | 完全自动执行，无需确认 | 回归测试、CI/CD 集成 |

### 执行模式

| 模式 | 说明 | 特点 |
|------|------|------|
| `exploration` | 探索模式 | LLM 实时分析页面结构，智能定位元素。适合首次执行或页面变化较大时 |
| `regression` | 回归模式 | 基于首轮执行记录的元素定位执行。速度快，适合稳定页面的回归测试 |
| `hybrid` | 混合模式 | 优先使用记录的元素定位，当元素未找到时自动切换到 LLM 智能适应 |

### 录制模式

| 模式 | 录制内容 | 存储开销 | 适用场景 |
|------|----------|----------|----------|
| `none` | 不录制 | 0 | 性能测试、快速执行 |
| `minimal` | 最终截图+操作日志 | ~150KB | 轻量回归 |
| `standard` ⭐ | 每步截图+快照 | ~1.5MB/10步 | 推荐日常使用 |
| `full` | 完整视频+详细日志 | ~50MB/10步 | 问题复现、调试 |

### 数据输入模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `preset` | 从环境变量或配置读取 | 自动化回归、CI/CD |
| `prompt` ⭐ | 运行时暂停询问用户 | 探索测试、首次录制 |
| `generate` | LLM 自动生成测试数据 | 批量测试、数据驱动 |

### 混合模式 (Script + 自然语言)

当业务流程或数据发生变化时，无需重新录制整个脚本，可使用混合模式灵活应对：

**脚本中插入自然语言步骤：**
```yaml
steps:
  - action: navigate
    url: "https://example.com/login"
  
  - action: type
    selector_hint: "用户名"
    text: "{{username}}"
  
  # ✨ 自然语言步骤：处理动态流程
  - action: natural_language
    description: "如果出现验证码，输入验证码并点击确认"
  
  # ✨ 条件分支
  - action: conditional
    condition: "如果页面显示'新用户引导'"
    then_steps:
      - action: natural_language
        description: "跳过引导，点击'跳过'按钮"
```

**运行时覆盖数据：**
```bash
# 覆盖占位符值
playwrights-pen run-script test.yaml --set username="new@test.com" --set password="NewPass"
```

### 模块化执行（案例复用）

将常用操作封装为可复用模块：

```yaml
# 定义登录模块
modules:
  login:
    description: "通用登录流程"
    parameters: [username, password]
    steps:
      - action: navigate
        url: "{{base_url}}/login"
      - action: type
        selector_hint: "用户名"
        text: "{{username}}"
      - action: type
        selector_hint: "密码"
        text: "{{password}}"
      - action: click
        selector_hint: "登录按钮"

# 在用例中复用模块
test_cases:
  - name: "下单测试"
    steps:
      - action: use_module
        module_name: login
        module_params:
          username: "test@example.com"
          password: "password123"
      - action: click
        selector_hint: "商品列表"
```

**智能跳过：** 如果当前浏览器已处于登录状态，模块可自动跳过。

### 数据提取与传递

跨步骤传递数据：

```yaml
steps:
  - action: click
    selector_hint: "提交订单"
  
  # 提取订单号保存到上下文
  - action: extract
    selector_hint: "订单号"
    save_as: order_id
  
  # 后续步骤使用提取的数据
  - action: type
    selector_hint: "搜索框"
    text: "{{order_id}}"
```

### 数据库检查

验证数据落库情况：

```yaml
steps:
  - action: click
    selector_hint: "提交订单"
  
  # 数据库断言
  - action: db_assert
    db_query: "SELECT status FROM orders WHERE id = '{{order_id}}'"
    db_column: status
    db_expected: "COMPLETED"
  
  # 数据库查询并保存结果
  - action: db_query
    db_query: "SELECT total FROM orders WHERE id = '{{order_id}}'"
    save_as: order_total
```

**数据库配置：**
```env
# .env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_USER=testuser
DB_PASSWORD=testpass
```

## 常见问题

### Q: 如何编写好的测试描述？

**好的测试描述应该：**
- 清晰描述每个操作步骤
- 使用易于理解的元素描述（如"搜索框"、"登录按钮"）
- 包含预期结果的验证

**示例：**
```
✅ 好的描述：
"打开淘宝首页，在搜索框输入'iPhone'，点击搜索按钮，验证页面显示商品列表"

❌ 不好的描述：
"测试搜索功能"
```

### Q: 测试失败时如何排查？

1. **检查浏览器窗口**：设置 `BROWSER_HEADLESS=false` 查看实际执行过程
2. **查看会话详情**：使用 `list-sessions` 命令查看失败步骤
3. **检查 LLM 配置**：确认 API Key 和模型配置正确
4. **验证网络连接**：确保能访问目标网站和 LLM API

### Q: 如何在 CI/CD 中使用？

```bash
# 使用无头模式和完全自动执行
playwrights-pen run "测试描述" --headless --confirm none --mode regression
```

### Q: 支持哪些浏览器？

目前主要支持 **Chromium**。Playwright MCP 同时支持 Firefox 和 WebKit，但推荐使用 Chromium 以获得最佳兼容性。

### Q: 数据存储在哪里？

所有数据存储在 `DATA_DIR` 配置的目录下（默认为 `./data`）：
- `testcases/` - 测试用例 JSON 文件
- `sessions/` - 执行会话 JSON 文件
- `snapshots/` - 页面快照文件

> **说明**：这些子目录会在首次访问时自动创建，无需手动初始化。
